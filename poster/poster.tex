  \begin{block}{Motivation \& Problem}

    Centralized exchanges (CEXs) provide deep liquidity and low-latency order books, while
    decentralized exchanges (DEXs) execute on-chain via AMMs with self-custody and transparency.
    These structural differences create measurable gaps in \textbf{price discovery}, \textbf{execution quality},
    and \textbf{transaction costs}.

    \vspace{0.6em}
    \textbf{Goal:} Build an end-to-end system that monitors CEX and DEX venues in real time and quantifies:
    deviation dynamics, lead-lag behavior, and slippage characteristics.

  \end{block}

  \begin{block}{System Overview}

    \textbf{Pipeline:} Two real-time streams run in parallel: (i) CEX tickers $\rightarrow$ price index, and (ii) DEX swaps from on-chain events. Both are stored, then joined to compute deviations and run analytics.

    \vspace{0.6em}
    \begin{figure}
      \centering
      \includegraphics[width=0.92\linewidth,height=15.0cm,keepaspectratio]{charts/structure.png}
      \caption{High-level architecture (schematic).}
    \end{figure}

  \end{block}

  \begin{block}{Price Index (CEX Reference Price)}

    We compute a robust CEX reference price by aggregating best bid/ask quotes from 8 exchanges.
    Each exchange contributes with a weight \(w_i(t)\) that is \textbf{adapted via EMA} based on
    observed top-of-book liquidity share (using bid/ask quantities).

    \vspace{0.2em}
    \[
      \textbf{price\_index}(t)=\frac{\sum_i \text{mid}_i(t)\cdot w_i(t)}{\sum_i w_i(t)},
      \qquad \text{mid}_i(t)=\frac{\text{bid}_i(t)+\text{ask}_i(t)}{2}
    \]

    In addition to the composite mid-price, we record \textbf{side VWAPs} and aggregated bid/ask quantities
    (\texttt{bid\_vwap}, \texttt{ask\_vwap}, \texttt{bid\_qty\_total}, \texttt{ask\_qty\_total}) for diagnostics.

  \end{block}

  \begin{block}{Analysis Metrics (Deviation, Lead-lag, Slippage)}

    \heading{1) Price Deviation (per DEX swap)}
    \[
      \textbf{deviation}=\frac{\text{DEX\_price}-\text{CEX\_index}}{\text{CEX\_index}}\times 100\%
    \]

    \heading{2) Lead-lag (price discovery)}
    Using log returns \(r(t)=\log p(t)-\log p(t-1)\), we compute cross-correlation over lags \(\tau\):
    \[
      \textbf{CCF}(\tau)=\text{Corr}\big(r_{\text{CEX}}(t),\; r_{\text{DEX}}(t+\tau)\big)
    \]

    \heading{3) Slippage (Uniswap V2 theoretical)}
    For reserves \((x,y)\) and input \(\Delta x\):
    \[
      \Delta y = y-\frac{xy}{x+\Delta x},
      \qquad
      \textbf{slippage}=\left|1-\frac{\Delta y/\Delta x}{y/x}\right|
    \]

  \end{block}

  \begin{block}{Takeaways}
    \begin{itemize}
      \item CEXs provide a fast reference price; DEX execution quality depends strongly on pool liquidity.
      \item Cross-venue deviation and lead--lag measurements quantify price discovery and convergence.
      \item A unified pipeline enables reproducible monitoring and retrospective analysis.
